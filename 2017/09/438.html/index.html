于 9 月 22 日买了个新电脑，发现我原来在 iMac 和 Thinkpad 上弄的那些东西，又需要重新安装配置在新电脑上，非常麻烦。之前看到别人为自己配置 HEXO 主题记录的小日志，觉得很方便。为了避免以后购入新电脑后出现这样一个阶段，在此对这台电脑上进行的折腾进行记录。

基本

Shell

OS

** 显示隐藏文件 ** - 2017-09-24 * permanently: defaults write com.apple.finder AppleShowAllFiles -boolean true ; killall Finder * temporarily: cmd+shift+.

** 阻止远程生成.DS_store 文件 ** - 2017-11-01 * defaults write com.apple.desktopservices DSDontWriteNetworkStores true

** 全局启用 True 灰度抗锯齿 / False 次像素抗锯齿渲染 ** - 2019-05-29 * defaults write -g CGFontRenderingFontSmoothingDisabled -bool False

** 调整字体抗锯齿级别 ** - 2019-05-29 * defaults -currentHost write -globalDomain AppleFontSmoothing -int 3

Touch ID - 2019-05-29 * sudo sed -i ".bak" '2s/^/auth sufficient pam_tid.so\'$'\n/g' /etc/pam.d/sudo

Homebrew

Install - 2017-09-22 * /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

Install - 2017-09-24 * via Homebrew brew install zsh zsh-completions

Oh My Zsh

Zsh 主题配置 ** 配置 ** - 2017-09-24 * ln -s ~/Dropbox/CONFIG/Oh-My-ZSH/.zshrc ~/.zshrc * Install the patched fonts from Vim-Powerline. Set both the Regular font and the Non-ASCII Font in “iTerm2 > Preferences > Profiles > Text” to use 14pt DejaVu Sans Mono for Powerline. * In “iTerm2 > Preferences > Profiles > Colors”, load preset Solarized Dark

Install - 2017-09-24 * via curl sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

hexo

** 安装插件 ** - 2018-10-10 * npm install hexo-renderer-pandoc --save * npm install hexo-symbols-count-time --save * npm install hexo-footnotes --save * npm install hexo-image-caption --save

** 添加 rss** - 2017-09-29 * npm install hexo-generator-feed --save

** 安装插件 ** - 2017-09-26 * npm install hexo-deployer-git --save * npm i --save hexo-wordcount

** 配置 ** - 2017-09-24 * ln -s ~/Dropbox/CONFIG/Hexo/_config.yml ~/myblog/_config.yml * ln -s ~/Dropbox/CONFIG/Hexo/source ~/myblog/source

Install - 2017-09-24 * via npm npm install hexo-cli -g hexo init myblog
